Slackrepo is a highly automated SlackBuild package builder. It builds packages from an input repository of SlackBuilds (for example, a clone of SlackBuilds.org), and then outputs them to a package repository.

This is a new, experimental tool for experienced Slackware users to manage their own package repositories. If you don't build your own packages, you don't need slackrepo. Most users will find that the existing popular tools are much better for their needs: sbopkg and sqg, AST, src2pkg, mkslack, etc.

For best results, slackrepo is intended to be run on a "clean" standard Slackware environment (like a "build box" or a virtual machine), with a full installation of Slackware and no additional packages (or very few). You can then use the package repository to install the packages on other hosts (e.g. with slackpkg+). However, slackrepo can also be used as a simple command-line host-based package builder and installer.

Some notable features are:

  • automatic updating - revisions of the SlackBuild repository determine updates and rebuilds of packages
  • clean building - dependencies are built as a 'tree', not a linear 'queue', and packages that are out-of-date with respect to their dependencies are rebuilt
  • quality assurance testing - optional tests can be performed on the SlackBuild files and built packages
  • no interaction required - highly automated and suitable for bulk testing
  • can build from SlackBuilds.org (SBo), Mate SlackBuilds (msb) and Cinnamon SlackBuilds (csb)
